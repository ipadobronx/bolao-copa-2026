# Feature 19 — Página 404 customizada

**Data:** 2026-05-07
**Status:** Aprovado para implementação inline
**Autor:** Spec direto (escopo trivial — página estática)
**Próximo passo:** execução inline → PR squash final

---

## 1. Contexto

Décima nona feature do `CLAUDE.md` §5 (extensão pré-lançamento). Sistema em produção em `https://malanacopa.com.br`. F1–F18 mergeadas. Hoje, qualquer rota inválida (`/foo`, `/comprar/[id-inexistente]/pix`, etc) cai num 404 básico (entregue na F1 setup) com layout simples — número + frase + 1 link. Não tem identidade nem CTA forte.

F19 entrega 404 alinhado com o branding "Mala na Copa": headline com tom de futebol, CTAs duplos pra landing e dashboard, robots noindex.

---

## 2. Decisões

| # | Pergunta | Escolha |
|---|----------|---------|
| Q1 | Sobrescrever 404 atual ou criar novo? | **Sobrescrever** `app/not-found.tsx`. Versão F1 era placeholder. |
| Q2 | Btn-outline ou btn-secondary? | **btn-secondary**. Projeto não tem `btn-outline`; `btn-secondary` em `globals.css` é o equivalente outline (border + texto). |
| Q3 | Fundo full-screen — declarar `bg-bg-dark` ou herdar do body? | **Declarar defensivo no `<main>`.** Body já tem em `globals.css @layer base`, mas `not-found.tsx` na raiz do `app/` não roda dentro de `(public)/layout.tsx` — declarar previne flash branco. |
| Q4 | Headline copy | **"Esse jogo não tá na tabela"** + "A página que você procurou foi pro chuveiro mais cedo. Mas a Copa 2026 segue rolando aqui." Tom de futebol consistente com landing. |
| Q5 | CTA secundário aponta pra `/dashboard` mesmo deslogado? | **Sim.** Middleware redireciona pra `/login` se não autenticado — UX correta. Quem está logado cai direto no dashboard. |
| Q6 | metadata robots | **`{ index: false, follow: false }`** — padrão pra 404. |
| Q7 | Tests automatizados | **Não.** Página estática sem lógica. Visual review em `pnpm dev` basta. |

---

## 3. Arquivo

`app/not-found.tsx` — substitui versão atual de 13 linhas por ~45 linhas com:

- `<main>` full-screen `min-h-screen bg-bg-dark flex items-center justify-center`
- Número "404" gigante: `font-display text-accent text-8xl md:text-[200px] leading-none`
- Headline em duas partes: "Esse jogo " + `<span class="text-accent">não tá na tabela</span>`
- Subtexto cinza secundário
- Dois CTAs em coluna no mobile, lado a lado no desktop:
  - Primário (amarelo): "Voltar pra landing" → `/`
  - Secundário (outline): "Ir pro dashboard" → `/dashboard`

---

## 4. Metadata

```ts
export const metadata = {
  title: 'Página não encontrada',
  description: 'A página que você procurou não existe.',
  robots: { index: false, follow: false },
};
```

Title resolverá pelo template global F18: `Página não encontrada · Mala na Copa`.

---

## 6. Fora de escopo

- Não vamos animar o número 404 (overhead JS desnecessário).
- Não vamos buscar bilhete/sessão pra personalizar o subtexto. Página tem que funcionar deslogada.
- Não vamos criar página `error.tsx` específica pra 500 — já existe em `app/error.tsx` (F1).
